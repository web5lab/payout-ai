Payout Distribution Flowchart

Start: User Buys Token from Offering Contract

User acquires tokens from a designated offering contract

Record Investment in Subgraph

At the time of investment, the user's wallet address and investment details are recorded in the subgraph.

User Holds Token for Earning Period

User holds the tokens for a specified duration (e.g., 1 year) to be eligible for yearly APY rewards. Transferring tokens to another wallet makes the user ineligible.

Start: Payout Distributor Initiates Distribution

The payout distributor decides to initiate a reward distribution for a specific earning period.

Has User Transferred Tokens from Original Wallet?
NO (No Transfer)
Distributor Checks Eligible Holders

The distributor uses the Verifier API to identify all users who met the holding criteria for the period and have not transferred tokens.

YES (Transferred)
End: User Ineligible for APY Reward

User is ineligible for APY rewards due to transferring tokens from the original investment wallet.

Distributor Calculates Reward Amounts

Based on the APY, holding duration, and verified token amounts, the distributor calculates the reward for each eligible holder.

Distributor Submits Total Amount to Payout Contract

The distributor transfers the total calculated reward amount to the designated payout smart contract on the blockchain.

Off-chain API Generates Claim Signature

For each eligible user, an off-chain API generates a unique digital signature, authorizing them to claim their specific reward amount from the payout contract.

User Requests to Claim Reward

The user initiates a claim request, providing their address and the off-chain signature received from the API.

Is the Off-chain Signature Valid and Claim Unused?
NO (Invalid/Used)
End: Claim Rejected

The payout contract rejects the claim due to an invalid or already used signature.

YES (Valid)
Payout Contract Transfers Reward to User

The payout smart contract verifies the signature and transfers the entitled reward tokens to the user's wallet.

End: User Successfully Claims Reward

The user receives their earned tokens, completing the payout cycle.
